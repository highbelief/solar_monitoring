# 520kW급 태양광 발전소 모니터링 시스템

본 프로젝트는 대학교 캡스톤 디자인 과목의 일환으로 개발된 **520kW급 태양광 발전소 모니터링 시스템**입니다. 이 시스템은 발전소의 현재 발전량과 예측 발전량을 실시간으로 모니터링하고, 관리자에게 필요한 정보를 시각화하여 제공합니다. Flask 프레임워크와 Chart.js를 사용하여 웹 기반으로 구축되었습니다.

> ⚠ **주의**: 본 프로젝트는 상업적 이용이 불가능하며, 이 프로젝트를 사용할 경우 발생하는 모든 책임은 사용자 본인에게 있습니다.

---

## 프로젝트 개요

이 시스템은 발전소 관리자가 태양광 발전소의 발전 현황을 실시간으로 확인하고, 향후 발전량을 예측할 수 있도록 돕습니다. 데이터베이스에는 발전소의 과거 발전량과 장비 상태가 저장되며, Flask 서버는 이 데이터를 가져와 실시간으로 시각화합니다. 예측 데이터를 표시하여 발전소 관리와 운영에 필요한 정보를 제공합니다.

---

## 주요 기능

- **실시간 발전량 모니터링**: 최근 발전량을 실시간 그래프로 표시하여, 최신 데이터를 시각적으로 확인할 수 있습니다.
- **예측 발전량 수치 표시**: 예측된 발전량, 배터리 상태, 장비 상태를 수치로 보여주어 미래의 발전량을 미리 파악할 수 있습니다.
- **데이터 로그 조회**: 발전소의 관측 및 예측 데이터를 한눈에 확인할 수 있는 로그 페이지를 제공합니다.
- **사이드바 메뉴**: 대시보드와 로그 페이지를 손쉽게 이동할 수 있도록 사이드바 메뉴를 제공합니다.

---

## 기술 스택

- **백엔드**: Flask (Python)
- **프론트엔드**: HTML, CSS, JavaScript, Chart.js (그래프 라이브러리)
- **데이터베이스**: MySQL
- **기타**: SQLAlchemy (ORM), Jinja2 (템플릿 엔진)

## 프로젝트 구조

```
solar_monitoring/
│
├── app.py                  # Flask 애플리케이션의 메인 파일
├── config.py               # 데이터베이스 및 기타 설정 파일
├── models.py               # 데이터베이스 테이블 모델 정의 파일
├── routes/                 # 페이지별 라우트를 정의하는 폴더
│   ├── __init__.py         # 블루프린트 초기화 파일
│   ├── dashboard.py        # 대시보드 페이지 라우트
│   ├── logs.py             # 로그 페이지 라우트
│   ├── user_management.py  # 유저 관리 페이지 라우트
│   └── pcs_control.py      # PCS 제어판 페이지 라우트
│
├── templates/              # HTML 템플릿 파일 저장 폴더
│   ├── dashboard.html      # 대시보드 페이지 템플릿
│   ├── logs.html           # 로그 페이지 템플릿
│   ├── user_management.html# 유저 관리 페이지 템플릿
│   └── pcs_control.html    # PCS 제어판 페이지 템플릿
│
├── static/                 # 정적 파일 저장 폴더 (CSS, JS, 이미지 등)
│   ├── dashboard.css       # 대시보드 페이지용 CSS 파일
│   ├── logs.css            # 로그 페이지용 CSS 파일
│   ├── user_management.css # 유저 관리 페이지용 CSS 파일
│   └── pcs_control.css     # PCS 제어판 페이지용 CSS 파일
│
└── requirements.txt        # 필요한 패키지 목록

```

### 파일 설명

- `app.py` - Flask 애플리케이션을 초기화하고 블루프린트를 등록하는 메인 파일입니다.
- `config.py` - MySQL 데이터베이스 설정 및 기타 설정이 포함된 파일입니다.
- `models.py` - 데이터베이스 테이블(관측 데이터 및 예측 데이터)을 정의한 파일입니다.
- `routes/dashboard.py` - 대시보드 페이지를 담당하는 라우트로, 실시간 그래프와 예측 데이터를 제공합니다.
- `routes/logs.py` - 로그 페이지를 담당하는 라우트로, 관측 및 예측 데이터를 표로 제공합니다.
- `templates/dashboard.html` - 대시보드 페이지의 HTML 템플릿으로, 실시간 그래프와 예측 수치가 표시됩니다.
- `templates/logs.html` - 로그 페이지의 HTML 템플릿으로, 관측 데이터와 예측 데이터를 표로 표시합니다.
- `requirements.txt` - 프로젝트 실행에 필요한 모든 패키지 목록입니다.

## 설치 및 실행 방법

### 1. 패키지 설치

먼저, `requirements.txt` 파일에 정의된 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

### 2. 데이터베이스 설정

MySQL에 새로운 데이터베이스를 생성하고, `config.py` 파일에서 해당 데이터베이스의 URI를 설정합니다.

```python
# config.py 예시
class Config:
    SQLALCHEMY_DATABASE_URI = 'mysql+pymysql://username:password@localhost/SolarDB'
    SQLALCHEMY_TRACK_MODIFICATIONS = False
```

### 3. 데이터베이스 테이블 생성

`app.py` 파일을 실행하여 데이터베이스에 테이블을 생성합니다.

```bash
python app.py
```

### 4. 서버 실행

서버를 실행하여 로컬 환경에서 프로젝트를 테스트합니다.

```bash
python app.py
```

서버가 성공적으로 실행되면, 웹 브라우저에서 다음 주소로 접속하여 시스템에 접근할 수 있습니다.

```
http://127.0.0.1:5000
```

## 사용 예시

### 1. 대시보드 페이지

- **현재 발전량 그래프**: 최근 발전량 데이터를 실시간으로 시각화하여 그래프로 표시합니다. 그래프는 최신 데이터를 우측에 표시하도록 구성되어 있습니다.
- **예측 발전량 수치**: 발전소의 예측된 발전량, 배터리 상태, 장비 상태를 그래프 옆에 수치로 표시합니다.
- **사이드바 메뉴**: 좌측 사이드바에서 대시보드와 로그 페이지로 손쉽게 이동할 수 있습니다.

### 2. 로그 페이지

- **관측 데이터와 예측 데이터를 표로 표시**: 데이터베이스에 저장된 모든 관측 및 예측 데이터를 표 형식으로 한눈에 확인할 수 있습니다.
- **사이드바 메뉴**: 대시보드 페이지와 마찬가지로 좌측 사이드바에서 대시보드와 로그 페이지로 이동할 수 있습니다.
